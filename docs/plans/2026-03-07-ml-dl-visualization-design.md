# ML/DL 視覺化工具教材系統 - 設計文件

> 日期：2026-03-07
> 計畫：115-18 ML/DL 視覺化工具 -- 編纂教材
> 提案人：楊東偉

## 1. 專案目標

建置一套服務於大專生的「ML/DL 視覺化互動教學系統」，以 18 週完整課綱串接機器學習至深度學習概念，兼顧理論與實作。

## 2. 技術決策

| 決策 | 選擇 | 理由 |
|------|------|------|
| 前端框架 | React + TypeScript | 生態系成熟、教學資源豐富 |
| 後端框架 | FastAPI (Python) | 非同步支援佳、與 ML 生態整合 |
| LLM API | 多模型抽象層 (Claude + GPT + 本地模型) | 彈性高、避免供應商鎖定 |
| 教材語言 | 中英雙語 | 講義中文、專有名詞附英文、程式碼英文註解 |
| 架構模式 | 模組化分層架構 | 漸進式導入、各模組獨立更新 |

## 5. 每週教學設計模板

每週 90 分鐘（可調整）：
- 前 30 分鐘：核心概念與假設（理論講解）
- 中間 40 分鐘：視覺化操作任務（平台實作）
- 後 20 分鐘：討論與反思 + 形成性評量

## 6. LLM 助教設計

### 多模型抽象層
- 統一介面支援 Claude API / OpenAI API / 本地模型（Ollama）
- Provider 切換不影響上層邏輯

### 分層提示策略 (Hint Ladder)
1. 先提問釐清學生的問題
2. 提示相關概念
3. 提示解題步驟
4. 最後才給局部範例

### 作業模式
- 引導式回覆，強制學生先提交思路或中間結果
- 不直接給出作業答案
- 結合平台學習紀錄做個人化提示

## 7. 學習分析儀表板

- 答題時間追蹤
- 錯誤型態分類
- LLM 查詢主題熱點
- 學習歷程數據視覺化
- 形成性/總結性評量結果

## 8. 評量設計

- 形成性：每週小作業、即時測驗、LLM 對話任務
- 總結性：期中/期末專題（報告、程式、展示）
- 評分面向：正確性 / 過程 / 反思

## 9. 資安與倫理

- 不上傳個資，資料匿名化
- API 金鑰以伺服端保管
- 教導 AI 倫理與負責任使用
