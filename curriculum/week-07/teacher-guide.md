## 教學策略與提示

### 引入策略 — 「20 Questions」類比
- 開場可以玩一個小遊戲：讓學生想一個動物，教師用 Yes/No 問題猜出來
- 引導：「你們剛才做的事情，就是人類版的決策樹！每一個問題就是一個分裂節點。」
- 這能直覺地讓學生理解「依特徵分裂」的概念

### 分裂準則教學
- **不要只講公式**，先用具體數字範例算一遍
- 準備一個簡單的 2 特徵、3 類別的資料集（可用 6-8 筆資料），讓學生手動計算 Gini 和 Entropy
- 強調：「Gini 和 Entropy 通常給出幾乎相同的結果，別花太多時間糾結哪個更好」

### 集成學習教學
- **Bagging 的類比：** 「想像你要做一個重要的決定，你會問多個朋友的意見，最後採取多數決」
- **Boosting 的類比：** 「想像你在準備考試，第一次模考錯的題目，第二次特別加強練習；第二次還是錯的，第三次更加重練」
- 這兩個類比能幫助學生快速抓住核心差異

### 常見迷思 Common Misconceptions

| 迷思 | 正確觀念 | 如何糾正 |
|------|---------|---------|
| 「決策樹不需要超參數調校」 | 未剪枝的樹幾乎必然過擬合 | Demo：不同 max_depth 的準確率差異 |
| 「樹越多越好」 | RF 確實如此（邊際效益遞減），但 GBDT 可能過擬合 | 實驗：GBDT 加太多樹後測試準確率下降 |
| 「GBDT 永遠比 RF 好」 | 在雜訊多的資料上 RF 可能更穩健 | 對比實驗：加入雜訊後的效能變化 |
| 「基尼和熵差很多」 | 實務上差異非常小 | 讓學生跑兩棵樹比較 |
| 「集成學習只是投票」 | Boosting 是加權累加殘差，不是投票 | 用殘差表格逐步演示 |

### 視覺化重點
- **決策邊界 (Decision Boundary)：** 這是本週最直覺的視覺化，讓學生看到 max_depth 如何影響邊界複雜度
- **樹結構圖：** `plot_tree()` 的結果要大圖展示，講解每個節點的資訊
- **學習曲線：** 對比 RF 和 GBDT 的學習曲線，讓學生「看到」偏差-變異的差異

## 學生可能遇到的問題與解法

### 技術問題

| 問題 | 解法 |
|------|------|
| `plot_tree()` 圖太擠看不清 | 使用 `figsize=(25, 15)` 加大圖面，或限制 `max_depth=3` 只顯示前幾層 |
| GBDT 訓練很慢 | 先用小的 n_estimators（如 50），確認程式正確後再跑大的 |
| 決策邊界圖只能畫 2D | 提示學生只選 2 個特徵，或使用 PCA 降到 2 維 |
| `oob_score` 報錯 | 確認 `bootstrap=True`（預設），且 n_estimators 夠大 |
| 中文字型問題 | 提供字型設定程式碼：`plt.rcParams['font.sans-serif'] = ['Microsoft JhengHei', 'SimHei']` |

### 概念問題

| 問題 | 引導方式 |
|------|---------|
| 「為什麼 Bootstrap 取樣是有放回的？」 | 引導思考：如果無放回抽取 N 筆，每次得到的就是原始資料，沒有差異性 |
| 「為什麼 RF 每次分裂只看部分特徵？」 | 引導：如果每棵樹都看所有特徵，最重要的特徵永遠被選中，樹就會很相似（高相關性） |
| 「殘差學習為什麼有效？」 | 用考試類比：你考 80 分，殘差是你離 100 分差的那 20 分，新的學習聚焦在這 20 分 |
| 「學習率為什麼要小？」 | 類比學走路：小步慢走更穩，不容易摔倒（過擬合） |

## 差異化教學建議

### 對進度較快的學生
- 引導嘗試 XGBoost/LightGBM 的安裝與使用
- 討論 GBDT 的數學推導（二階泰勒展開）
- 嘗試在 Kaggle 的 Titanic 或 Housing Prices 資料集上應用
- 探討特徵重要度的不同計算方式（MDI vs Permutation Importance）

### 對進度較慢的學生
- 確保至少理解決策樹的基本建構（分裂 → 停止 → 預測）
- 集成學習先聚焦在「多個模型比一個好」的直覺，不必深入數學
- 提供部分完成的程式碼模板，讓學生填空
- 一對一指導 `plot_tree()` 的解讀

### 小組活動建議
- **手動建樹活動（10 分鐘）：** 給 4-5 組學生各一份小型資料集（10 筆，2 特徵），讓他們手動選擇分裂點，比較各組結果
- **模型選擇辯論（5 分鐘）：** 將學生分為「RF 隊」和「GBDT 隊」，各自陳述為何自己的方法更好

## 與前後週的銜接

### 與前週的連結（Week 6: SVM）
- 對比：SVM 的決策邊界是超平面（線性或核映射後），決策樹的邊界是軸對齊的矩形
- 引導：「SVM 用距離最大化做決策，決策樹用純度最大化做決策——不同的哲學」
- 可以展示相同資料集上 SVM 和 Decision Tree 的決策邊界差異

### 與下週的連結（Week 8: Feature Importance & SHAP）
- 本週介紹的特徵重要度是基礎，下週會更深入
- 預告：「本週我們知道哪些特徵重要，但不知道它們如何影響預測——這就是 SHAP 要解決的問題」
- 本週的 RF/GBDT 模型將直接用於下週的 SHAP 分析

### 即時檢核問題（課堂可用）

1. 「如果一個決策樹的訓練準確率是 100%，你覺得這是好事嗎？」（預期答案：過擬合）
2. 「Random Forest 的每棵樹都是好的分類器嗎？」（預期答案：每棵樹可能不完美，但集成後變好）
3. 「GBDT 的第 100 棵樹和第 1 棵樹，哪棵更重要？」（預期答案：第 1 棵，因為它處理了最大的殘差）
