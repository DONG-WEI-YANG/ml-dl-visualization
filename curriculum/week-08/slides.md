## Slide 4: 方法一 — 內建特徵重要度
### Built-in Feature Importance
- 基於不純度下降 (Impurity Decrease)
- 決策樹 / 隨機森林 / GBDT 內建支援

```python
model = RandomForestClassifier()
model.fit(X_train, y_train)
importances = model.feature_importances_
```

**限制：** 偏向高基數特徵、不考慮測試集表現

---

## Slide 5: 方法二 — 排列重要度
### Permutation Importance
**核心思想：** 打亂某特徵 → 效能下降多少 = 多重要

```
1. 計算基準效能
2. 打亂特徵 f 的值
3. 重新計算效能
4. 重要度 = 基準 - 打亂後
```

**優勢：** 模型不可知、基於測試集

## Slide 7: SHAP 的博弈論根源
### 從合作博弈到模型解釋

**博弈論類比：**
- 玩家 = 特徵
- 聯盟 = 特徵子集
- 報酬 = 模型預測值

**Shapley 值：** 每個玩家在所有可能加入順序中的平均邊際貢獻

Lloyd Shapley (1953) → 2012 年諾貝爾經濟學獎

---

## Slide 8: Shapley 值的直覺
### 一個簡單的例子

三人合作搬鋼琴，報酬 $600：
- A 一人搬不動 → $0
- B 一人搬不動 → $0
- A + B 能搬 → $400
- A + B + C 搬得快 → $600

**Shapley 值** 考慮所有可能的加入順序來公平分配！

---

## Slide 9: SHAP 公式
### 預測值的分解

$$f(x) = \phi_0 + \phi_1 + \phi_2 + \cdots + \phi_M$$

| 符號 | 意義 |
|:----:|:----:|
| $f(x)$ | 最終預測值 |
| $\phi_0$ | 基準值（平均預測） |
| $\phi_i$ | 特徵 $i$ 的 SHAP 值 |

**每個預測 = 基準值 + 各特徵的貢獻**

---

## Slide 10: Shapley 值的四大公理
### 為什麼 SHAP 是「公平」的？

1. **效率性 Efficiency** — 貢獻之和 = 總報酬
2. **對稱性 Symmetry** — 等價特徵得到等值
3. **虛擬性 Null Player** — 無用特徵得 0
4. **可加性 Additivity** — 可跨模型加總

Shapley 值是**唯一**同時滿足四公理的方法！

---

## Slide 11: SHAP 計算方法
### 針對不同模型的最佳化

| 方法 | 適用模型 | 速度 |
|:----:|:--------:|:----:|
| TreeSHAP | RF, GBDT, XGBoost | 極快 |
| KernelSHAP | 任意模型 | 較慢 |
| DeepSHAP | 神經網路 | 中等 |
| LinearSHAP | 線性模型 | 極快 |

**實務首選：** 樹模型 → TreeSHAP

## Slide 12: 蜂群圖 Beeswarm Plot
### 全域解釋的首選圖表

```
              ← 壓低預測    推高預測 →
特徵 A    🔵🔵🔵🔵 .... 🔴🔴🔴🔴
特徵 B    🔴🔵🔵 ..... 🔴🔴
特徵 C    🔵🔴 ...... 🔵🔴
```

- Y 軸：特徵（依重要度排序）
- X 軸：SHAP 值
- 顏色：紅 = 特徵值高，藍 = 特徵值低
- 每個點 = 一個樣本

---

## Slide 13: 蜂群圖解讀技巧
### 三種常見模式

**模式 1 — 正相關：** 紅點在右，藍點在左
→ 特徵值越高，預測越高

**模式 2 — 負相關：** 藍點在右，紅點在左
→ 特徵值越低，預測越高

**模式 3 — 複雜關係：** 紅藍混雜
→ 可能有非線性或交互作用

---

## Slide 14: 力圖 Force Plot
### 單一預測的故事

```
基準值 = 0.5
────────────────────────────────────→
  ◄── 年齡高 (+0.15)  收入低 (-0.08) ──►
                    最終預測 = 0.57
```

- 紅色箭頭：推高預測的特徵
- 藍色箭頭：壓低預測的特徵
- 向客戶解釋：「為什麼你的風險分數是 0.57？」

---

## Slide 15: 依賴圖 Dependence Plot
### 觀察特徵效果與交互作用

- X 軸：特徵的實際值
- Y 軸：該特徵的 SHAP 值
- 顏色：交互特徵的值

**可發現：**
- 線性 / 非線性關係
- 閾值效果（如收入超過某值後 SHAP 值驟變）
- 與其他特徵的交互作用

## Slide 16: 局部 vs 全域解釋
### Local vs Global Explanation

| | 局部 Local | 全域 Global |
|:---:|:---:|:---:|
| 問題 | 這筆預測為何如此？ | 模型整體行為？ |
| SHAP 圖 | 力圖、瀑布圖 | 蜂群圖、長條圖 |
| 場景 | 個案審查 | 模型審計 |

**SHAP 的優勢：局部 → 聚合 → 全域**

---

## Slide 17: LIME 快速介紹
### Local Interpretable Model-agnostic Explanations

1. 在目標樣本附近產生擾動資料
2. 用原始模型預測擾動資料
3. 訓練簡單模型（如線性回歸）近似局部行為
4. 簡單模型的係數 = 特徵重要度

**vs SHAP：** 更快但較不穩定，無理論公平性保證

---

## Slide 18: 透明度 vs 預測能力
### The Trade-off

```
準確率 ↑
  │      * DL
  │     * GBDT     ← 搭配 SHAP = 兩全其美
  │   * RF
  │  * 決策樹
  │ * 線性回歸
  └──────────→ 可解釋性
```

**打破兩難：** 複雜模型 + SHAP = 高效能 + 可解釋
