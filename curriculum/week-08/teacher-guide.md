## 教學重點 Key Points

1. **從第 7 週延伸：** 學生已會訓練隨機森林與 GBDT，本週聚焦在「理解模型的行為」，自然銜接
2. **博弈論不需過度深入：** Shapley 值的數學公式用直覺類比（搬鋼琴的例子）帶過即可，重點在理解「公平分配貢獻」的概念
3. **圖表解讀是核心技能：** 花最多時間在蜂群圖的解讀上，這是學生未來最常用的 SHAP 圖表
4. **實作優先於理論：** 讓學生先看到 SHAP 圖表的效果，再回頭理解理論會更有動力
5. **連結真實場景：** 多舉金融（信貸）、醫療（診斷）的例子，讓學生感受 XAI 的實際價值

## 教學策略 Teaching Strategies

### 引入動機 Motivation Hook
- 開場問學生：「如果銀行拒絕你的貸款申請，你會不會想知道原因？」
- 展示一個力圖，讓學生直覺感受 SHAP 的解釋力
- 提及歐盟 GDPR 的「解釋權」，說明這不只是學術議題

### Shapley 值的教學建議 Teaching Shapley Values
- **不要**一開始就秀公式，先用搬鋼琴的類比
- 用 3 個特徵的簡單例子手動計算一次 Shapley 值
- 強調「所有可能的加入順序」→「平均邊際貢獻」
- 四大公理用白話說明即可，不需要數學證明

### 圖表教學順序 Chart Teaching Order
1. **先教長條圖 (Bar Plot)** → 最簡單，類似排列重要度
2. **再教蜂群圖 (Beeswarm)** → 加入特徵值的顏色維度
3. **接著力圖 (Force Plot)** → 單一樣本的故事
4. **最後依賴圖 (Dependence)** → 深入單一特徵的效果

### 互動示範 Live Demo
- 在課堂上即時執行 Notebook，邊跑邊解釋
- 讓學生猜測：「你覺得哪個特徵最重要？」→ 再用 SHAP 驗證
- 故意選一個反直覺的例子（如模型依賴了不該依賴的特徵）→ 引發討論

## 常見問題與排除 Troubleshooting

### Q1: SHAP 安裝失敗
- 確認 Python 版本 >= 3.8
- 嘗試：`pip install shap --no-cache-dir`
- Windows 可能需要先安裝 Visual C++ Build Tools
- 備用方案：使用 Google Colab（已預裝 SHAP）

### Q2: SHAP 計算非常慢
- 確認使用 TreeExplainer（針對樹模型）而非 KernelExplainer
- 減少資料量：先用 `X_test[:100]` 測試
- KernelSHAP 可設定 `nsamples` 參數降低計算量

### Q3: 力圖無法在 Jupyter 中顯示
- 需要先執行 `shap.initjs()`
- 如果仍不顯示，改用 `matplotlib=True` 參數：
```python
shap.plots.force(shap_values[0], matplotlib=True)
```

### Q4: 學生混淆特徵重要度與 SHAP 值
- 強調：特徵重要度是**一個數字**（全域），SHAP 值是**每個樣本一個數字**（局部）
- 用表格對比兩者的概念

### Q5: 學生不理解 Shapley 值的公式
- 不用糾結公式！重點在理解概念
- 用 3 個特徵的例子手算一次就夠
- 強調 SHAP 套件會自動計算
