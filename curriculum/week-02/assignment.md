## 第一部分：Seaborn 統計圖表（20%）

### 要求

使用 Seaborn 繪製**至少 3 種不同類型**的統計圖表，並為每張圖撰寫簡要解讀。

### 具體任務

1. **分布圖（擇一或多）：** 使用 `histplot`（含 KDE）或 `kdeplot` 繪製至少一個數值變數的分布圖。
   - 加入 `hue` 參數進行分組對比
   - 在圖上標記均值 (Mean) 與中位數 (Median)

2. **關聯圖：** 使用 `pairplot` 繪製數值變數的成對關係圖。
   - 使用 `hue` 區分不同類別
   - 對角線使用 `kde` 顯示密度

3. **分類圖（擇一或多）：** 使用 `boxplot`、`violinplot`、或 `swarmplot` 繪製分類變數與數值變數的關係。
   - 使用 `hue` 進行分組比較
   - 在文字描述中說明箱型圖或小提琴圖所揭示的分布特性

4. **熱力圖：** 繪製相關係數矩陣的熱力圖。
   - 使用 `annot=True` 顯示數值
   - 選擇合適的 colormap（建議 `coolwarm` 或 `RdBu_r`）
   - 使用上三角遮罩 (mask) 避免重複資訊

### 交付物
- 包含至少 3 張 Seaborn 圖表的 Notebook cells
- 每張圖下方附 2-3 句解讀文字

## 第三部分：完整 EDA 流程（20%）

### 要求

對所選資料集進行**完整的四步驟 EDA**，展現系統性的分析方法。

### 具體任務

#### Step 1: 資料概覽 Data Overview
- 顯示資料集的形狀 (`shape`)
- 使用 `info()` 檢視欄位型態
- 使用 `describe()` 呈現描述統計量
- 列出缺失值數量與比例
- 用 1-2 句文字總結資料集的基本特性

#### Step 2: 單變數分析 Univariate Analysis
- 對**至少 2 個數值變數**繪製分布圖（直方圖 + KDE）
- 對**至少 1 個類別變數**繪製計數圖或長條圖
- 計算偏態 (Skewness) 與峰態 (Kurtosis)，並解釋分布的形狀
- 用文字說明是否發現異常分布

#### Step 3: 雙變數/多變數分析 Bivariate/Multivariate Analysis
- 繪製相關係數熱力圖，找出相關性最強的變數對
- 對目標變數（如 survived, tip, price）進行分組比較（至少 2 個分組因子）
- 使用散佈圖或 pairplot 探索變數間的關聯
- 用文字說明你發現的重要關聯

#### Step 4: 洞察摘要 Insights
- 以條列方式列出**至少 3 個**你從 EDA 中發現的洞察
- 每個洞察需有對應的視覺化證據支持

### 交付物
- 按四步驟組織的 Notebook，每步附 Markdown 說明
- 洞察摘要列表

## 第四部分：缺失值處理實作（20%）

### 要求

展示缺失值的偵測、視覺化與處理全過程。

### 具體任務

1. **缺失值偵測與視覺化：**
   - 計算每個欄位的缺失值數量與比例
   - 繪製缺失值矩陣（`sns.heatmap(df.isnull())`）或缺失比例長條圖
   - 用文字說明哪些欄位有缺失、缺失的嚴重程度

2. **缺失機制判斷：**
   - 對至少一個缺失欄位，分析缺失值與其他欄位的關係
   - 判斷可能的缺失機制（MCAR / MAR / MNAR）
   - 以 1-2 句文字解釋你的判斷依據

3. **處理策略實作：**
   - 實作**至少兩種**不同的填補方法（例如：中位數填補 + KNN 填補，或均值填補 + 分組填補）
   - 繪製填補前後的分布對比圖（在同一張圖中疊加）
   - 比較不同填補方法的效果，用文字說明你最終選擇哪種方法及原因

4. **（加分項）** 使用 `sklearn.impute.IterativeImputer` (MICE) 進行填補，並與簡單方法比較。

### 交付物
- 缺失值分析與處理的完整 Notebook cells
- 填補前後對比圖
- 方法選擇的理由說明

## 提示與注意事項 Tips & Notes

### 程式碼品質
- 每個 Cell 加入適當的 Markdown 說明
- 圖表需有標題 (`title`)、座標軸標籤 (`xlabel`, `ylabel`)、圖例 (`legend`)
- 使用有意義的變數名稱
- 適當的程式碼註解（中文或英文均可）

### 視覺化品質
- 選擇合適的圖表類型（參考講義的圖表選擇指南）
- 注意色彩的可讀性與無障礙性
- 避免過度裝飾，保持簡潔
- 圖表大小適當，文字清晰可讀

### 分析深度
- 不只描述「是什麼」，更要思考「為什麼」
- 發現異常時嘗試解釋可能的原因
- 提出後續可驗證的假設

### AI 助教使用
- 可向 AI 助教詢問圖表選擇建議或程式碼除錯
- 但作業分析與報告內容須為自己的原創見解
- AI 助教在作業模式下會要求你先提出思路
