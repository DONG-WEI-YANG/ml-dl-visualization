## 教學重點與策略 Teaching Strategies

### 1. 開場導入（5 分鐘）

**建議提問：**
- 「上週我們學了邏輯迴歸可以畫一條決策邊界。如果有很多條線都能分開資料，我們怎麼知道哪條最好？」
- 「想像你要在兩群人之間畫一條分隔線，你會怎麼畫？」

**銜接要點：**
- 從第 5 週的邏輯迴歸自然過渡到 SVM
- 強調 SVM 回答的是「哪條決策邊界最好」的問題
- 引入「間隔最大化」的直覺

### 2. SVM 直覺與間隔最大化（10 分鐘）

**核心類比 —— 最寬的馬路：**
- 兩群人（兩類資料）站在馬路兩邊
- 邏輯迴歸：找一條能分開的線（任何線都行）
- SVM：在兩群人之間鋪一條最寬的馬路
- 馬路邊緣碰到的人就是「支持向量」

**教學提示：**
- 先用直覺解釋，再引入數學
- 數學公式不需要推導，重點在理解含義
- $\min \frac{1}{2}\|\mathbf{w}\|^2$ → 告訴學生「最小化 w 的長度就是最大化間隔」
- 用圖形而非公式建立直覺

**常見學生疑問：**
- Q：「為什麼要最大化間隔？」 → A：「間隔越大，新資料落在模糊地帶的機率越小，泛化能力越好。」
- Q：「為什麼用 $y \in \{-1,+1\}$ 而不是 $\{0,1\}$？」 → A：「數學上可以把兩類的約束統一為一個不等式。」

### 3. 支持向量、硬/軟間隔、參數 C（10 分鐘）

**關鍵概念傳達：**

1. **支持向量的稀疏性：** 這是 SVM 最重要的特性之一。只有少數點決定模型，其他點完全不影響。可以用互動 Demo 拖曳非支持向量的點，讓學生看到邊界完全不動。

2. **硬→軟間隔的過渡：** 不要花太多時間在硬間隔上，因為實務中幾乎不用。重點是讓學生理解「為什麼需要軟間隔」（因為現實資料有噪音和重疊）。

3. **參數 C 的直覺：**
   - C = Cost of misclassification（錯誤分類的代價）
   - C 大 → 「嚴格老師」，不允許犯錯
   - C 小 → 「寬容老師」，允許一些小錯以換取更簡單的規則
   - **這是學生最容易混淆的參數！** 務必用視覺化 Demo 反覆強調

**互動活動建議：**
- 使用平台的滑桿調整 C 值，讓學生即時觀察邊界變化
- 讓學生預測：「如果我把 C 從 1 調到 100，邊界會變寬還是變窄？」

### 4. 核技巧（10 分鐘）

**這是本週最難的概念，需要格外注意教學策略。**

**建議的教學順序：**

1. **先展示問題：** 用 Moon 或 Circle 資料集，讓學生看到線性 SVM 完全無法分類
2. **提出思路：** 「如果我們把資料搬到更高維的空間呢？」
3. **3D 映射 Demo：** 用 Circle 資料集做 2D → 3D 映射的視覺化（Notebook 第 7 節）
4. **引入核技巧：** 「好消息是，我們不需要真的計算高維座標！」
5. **四種核函數：** 快速介紹，重點在 RBF

**教學陷阱 —— 避免以下情況：**
- 不要花太多時間在核技巧的數學推導上（對偶問題、KKT 條件等）
- 不要讓學生覺得需要「理解」無限維映射的具體形式
- 重點是：核技巧讓 SVM 可以處理非線性問題，RBF 是最常用的核

**如果學生問「映射到無限維是什麼意思？」：**
- 可以告訴他們：RBF 核等價於把每個資料點用無窮多個特徵來表示（各種階的泰勒展開項），但核函數讓我們不需要真的計算這些特徵
- 重要的不是「無限維」本身，而是「我們可以在不計算高維座標的情況下，得到高維空間中的分類結果」

### 5. gamma 參數與 C-gamma 搜索（實作中講解）

**教學重點：**
- gamma 控制「每個點的影響範圍」
- gamma 大 → 每個點只影響附近很小的區域 → 決策邊界複雜
- gamma 小 → 每個點影響很大的區域 → 決策邊界平滑
- C 和 gamma 必須**同時**調校，不能只看一個

**實務提示（告訴學生）：**
- 永遠先做特徵縮放！
- C 和 gamma 用對數尺度搜索
- 先粗搜再細搜
- 用交叉驗證，不要用訓練集準確率

| Notebook 節次 | 時間 | 教師角色 |

| 第 2 節：C 值實驗 | 5 min | 帶著做前半，後半自主 |
| 第 3 節：核函數比較 | 7 min | 帶著做，重點講解差異 |

| 第 5 節：C-gamma 熱力圖 | 8 min | 帶著做 GridSearchCV |

| 第 7 節：幾何直覺 | 5 min | 選做，帶有興趣的學生看 |

### 常見實作問題

1. **SVM 訓練很慢：**
   - 原因：資料量大或 gamma 太大
   - 解法：減少資料量或限制搜索範圍

2. **決策邊界繪製時出現警告：**
   - 通常是 contourf 的 levels 問題
   - 確保網格範圍涵蓋所有資料點

3. **特徵縮放忘記：**
   - SVM 對尺度極其敏感
   - 忘記縮放會導致一個特徵主導
   - 提醒學生：測試集要用 `transform()` 不是 `fit_transform()`

4. **GridSearchCV 太慢：**
   - 使用 `n_jobs=-1` 平行計算
   - 減少搜索網格的大小
   - 先用小資料集測試

## 討論引導 Discussion Facilitation

### 討論主題：SVM vs Logistic Regression

**引導問題：**

1. **入門級：** 「SVM 和邏輯迴歸都可以做分類，那它們的決策邊界有什麼不同？」
   - 預期回答：SVM 尋求最大間隔，LR 尋求最大似然
   - 深化：SVM 的邊界只由支持向量決定，LR 的邊界由所有資料點共同決定

2. **進階級：** 「在什麼情況下，SVM 比邏輯迴歸好？反過來呢？」
   - SVM 好的情況：非線性資料（核方法）、高維稀疏資料、小樣本
   - LR 好的情況：需要機率輸出、大量資料、需要可解釋性

3. **應用級：** 「如果你要做一個垃圾郵件分類器，你會選 SVM 還是 LR？為什麼？」
   - 引導學生考慮：文本資料的高維稀疏性、是否需要機率、計算資源

4. **思辨級：** 「核技巧可以把 SVM 變成非線性分類器。那 LR 能不能也做非線性分類？」
   - 引導到：手動特徵工程（加入交互項、多項式特徵）也能讓 LR 做非線性分類
   - SVM 的核技巧只是自動化了這個過程

### 開放式討論問題
- 「現在深度學習這麼流行，SVM 還有用嗎？」
  - 引導到：SVM 在小樣本、高維、可解釋性方面仍有優勢
  - 例如：基因組學、文本分類（傳統 NLP）、醫學影像小資料集

## 學生常見迷思 Common Misconceptions

| 迷思 | 正確理解 |
|------|---------|
| 「SVM 只能做二元分類」 | SVM 可以透過 OvO 或 OvR 策略做多類分類 |
| 「C 越大越好」 | C 太大會過擬合，需要用 CV 選擇合適的 C |
| 「RBF 核永遠最好」 | 高維稀疏資料用 Linear 通常更好；核的選擇取決於資料 |
| 「核技巧真的把資料映射到高維」 | 核技巧的重點是「不需要真的映射」，只計算內積 |
| 「SVM 不需要特徵縮放」 | SVM 基於距離計算，對尺度非常敏感，必須縮放 |
| 「支持向量越多越好」 | 支持向量太多說明模型可能太簡單（C 太小）或資料太混亂 |
| 「gamma 是 RBF 核的半徑」 | gamma 是影響範圍的倒數，gamma 越大影響範圍越小 |

### 延伸議題（如果時間允許）
- **SVM 與正則化的關係**：SVM 的 $\frac{1}{2}\|\mathbf{w}\|^2$ 等同於 L2 正則化
- **Hinge Loss 與 Cross-Entropy 的關係**：兩者都是 0-1 Loss 的凸上界
- **SVM 在深度學習時代的角色**：作為特徵提取後的分類器、小樣本學習
