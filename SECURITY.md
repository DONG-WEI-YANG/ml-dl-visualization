# 資安與隱私政策 / Security & Privacy Policy

> 對應設計文件第 9 節「資安與倫理」

## 1. 資料處理原則

- **不蒐集個資**：系統僅以匿名學生 ID 記錄學習事件，不蒐集姓名、身分證字號、電話等個人資料
- **資料匿名化**：所有學習分析均以匿名方式呈現，無法反向識別個別學生
- **最小蒐集原則**：僅蒐集教學所需之學習行為資料（答題、互動次數、學習時間）

## 2. API 金鑰管理

- 所有 API 金鑰（Anthropic、OpenAI）**僅存放於伺服端 `.env` 檔案**
- `.env` 已列入 `.gitignore`，不會被提交至版本控制
- 前端不接觸任何 API 金鑰，所有 LLM 呼叫透過後端代理

## 3. 身份驗證與授權

| 機制 | 說明 |
|------|------|
| 密碼雜湊 | PBKDF2-HMAC-SHA256，100,000 次迭代，隨機鹽值 |
| JWT Token | HMAC-SHA256 簽章，預設 8 小時過期 |
| 角色權限 | admin / teacher / student 三級存取控制 |
| WebSocket | 連線時驗證 token 參數 |

## 4. 部署安全建議

- [ ] 修改預設管理員密碼（環境變數 `DEFAULT_ADMIN_PASSWORD`）
- [ ] 設定強 JWT 密鑰（環境變數 `JWT_SECRET`）
- [ ] 限制 CORS 來源為實際域名（環境變數 `CORS_ORIGINS`）
- [ ] 啟用 HTTPS（透過反向代理如 Nginx）
- [ ] 定期備份 SQLite 資料庫（`data/app.db`）

## 5. 速率限制

- 預設 60 次請求/分鐘/IP
- 保護登入端點免受暴力攻擊
- 保護 LLM API 端點免受濫用

## 6. 第三方服務

| 服務 | 用途 | 資料傳送 |
|------|------|---------|
| Anthropic Claude API | AI 助教 | 學生問題文字（不含個資） |
| OpenAI GPT API | AI 助教（備選） | 學生問題文字（不含個資） |
| Ollama (本地) | AI 助教（離線） | 不傳送任何資料 |
